1041 /* \example geometry/example1/geometry_example1.cpp
1042 
1043  Suppose that you frequently use the same complex geometry and you
1044  are tired of always having to include the long definition of this geometry
1045  into your input file. In this case it may be worthwhile to implement your
1046  own geometry DSO that constructs the geometry of interest with very little
1047  input (or even no input at all). This example illustrates how to do this
1048  programatically.
1049 
1050  As this is just an example, the geometry we will be constructing is not very
1051  complex: it consists of a cylinder inscribed in a sphere and the sphere
1052  inscribed in a box as shown in the figure ???. We want to be able to define
1053  the media of the cylinder, the sphere and the box in the input file
1054  (so that we can easily use different PEGS data sets, for example). All other
1055  dimensions of our example geometry are fixed.
1056 
1057  To create our new geometry, we make a new subdirectory in the egspp
1058  geometries subdirectory called \c example1. To be able to use the
1059  predefined rules for building geometry libraries we create a dummy
1060  header file \c %geometry_example1.h that does nothing else except to
1061  define the \c EGS_EXAMPLE1_DLL
1062  macro:
1063  \include geometry/example1/geometry_example1.h
1064  and put the implementation in \c %geometry_example1.cpp.
1065  \dontinclude geometry/example1/geometry_example1.cpp
1066  In this file we include the just created header file and the EGS_Input
1067  class header file to get access to its definition,
1068  needed in the \c %createGeometry() function (see below):
1069  \until egs_input
1070  We also need the declarations of the various geometry classes that we
1071  will use to construct our geometry:
1072  \until egs_envelope_geometry.h
1073  We then define the dimensions of the various geometrical structures
1074  \until box_size
1075 
1076  The remaining task is to implement a C-style \c %createGeometry() function
1077  exported by the DSO
1078  \until EGS_EXAMPLE1_EXPORT
1079  We check that the input object is valid and return \c null if it is not
1080  \until }
1081  We then look for a definition of the cylinder medium and print a warning if
1082  such a definition is missing:
1083  \skipline cyl_medium
1084  \until }
1085  We repeat basically the same code to obtain the sphere and the box
1086  media
1087  \until }
1088  \until }
1089  and return \c null if any of the media definition was missing:
1090  \until if
1091  Now we are ready to construct our geometry. We first create a set of
1092  parallel z-planes
1093  \until EGS_ZProjector
1094  As the planes will only be used by our geometry object internally, we don't
1095  care how the plane set is named and therefore give it a name using
1096  the static function EGS_BaseGeometry::getUniqueName().
1097  In a similar way we create a set of z-cylinders consisting of a single
1098  cylinder
1099  \until ZProjector
1100  and make a closed cylinder modeled as a 2D-geometry from the planes and the
1101  cylinder surface:
1102  \until getUniqueName
1103  We then set the medium of the cylinder
1104  \until setMedium
1105  The next step is to create the sphere as a set of spheres consisting of
1106  a single sphere
1107  \until getUniqueName
1108  and to fill it with the sphere medium defined in the input
1109  \until setMedium
1110  We then use an envelope geometry to inscribe the cylinder into the just
1111  created sphere:
1112  \until getUniqueName
1113  The next step is to create the outer box and fill it with the box medium
1114  defined in the input
1115  \until setMedium
1116  (as our geometry will always be centered about the origin, we pass a
1117  \c null affine transformation to the constructor of the box).
1118  We now inscribe the \c the_sphere geometry object (which is the sphere
1119  with the cylinder inscribed in it) into the just created box using
1120  again an envelope geometry,
1121  \until the_box
1122  set the name of \c the_box from the input using the inherited
1123  \link EGS_BaseGeometry::setName(EGS_Input*) setName() \endlink method,
1124  \until setName
1125  and return a pointer to the just created geometry
1126  \until }
1127  \until }
1128  That's all.
1129 
1130  In the Makefile we include the egspp make config files,
1131  \dontinclude geometry/example1/Makefile
1132  \skipline EGS_CONFIG
1133  \until my_machine
1134  set the C-preprocessor defines to the standard set of defines
1135  for our egspp configuration,
1136  \until DEFS
1137  define the name of our DSO,
1138  \until library
1139  and and the set of files needed to build the DSO
1140  \until lib_files
1141  Because we are using planes, cylinders, spheres, a ND-geometry, a box
1142  geometry and an envelope geometry to construct our geometry, we must
1143  link against the geometry libraries containing these geometries.
1144  We accomplish this by seting the \c link2_libs make variable to the
1145  list of needed libraries
1146  \until egs_genvelope
1147  We then include the standard rules for building a DSO
1148  \until include
1149  and set the dependencies of our object files
1150  \until make_depend
1151  We must also add the dependencies on the various geometry header files
1152  that we are including
1153  \until egs_envelope_geometry.h
1154  We can now build our geometry DSO by typing \c make and use
1155  it by including input like this in the input file
1156  \verbatim
1157  :start geometry:
1158  library = geometry_example1
1159  name = some_name
1160  cylinder medium = H2O521ICRU
1161  sphere medium = AL521ICRU
1162  box medium = AIR521ICRU
1163  :stop geometry:
1164  \endverbatim
1165  It is worth noting that we could have constructed this geometry as the
1166  \link EGS_UnionGeometry union \endlink of the cylinder pointed to
1167  by \c the_cyl, the sphere pointed to by \c sphere and the
1168  box pointed to by \c box. However, such an implementation would be
1169  much slower at run time because for each invocation of a geometry
1170  method the union will have to interogate the geometry methods of all
1171  3 objects. In contrast, the envelope geometry will be checking only
1172  against the box for points outside of the box, the box and the sphere for
1173  points inside the box but outside the sphere, the sphere and the cylinder
1174  for points inside the sphere but outside the cylinder and the cylinder
1175  only for points inside the cylinder.
